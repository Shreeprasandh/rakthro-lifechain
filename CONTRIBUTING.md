# CONTRIBUTING.md

## Contributing to RAKTHRO

Thank you for your interest in contributing to RAKTHRO. Follow these guidelines to ensure a smooth workflow.

### Setup
1. Fork the repository.
2. Create a feature branch: `git checkout -b feature/YourFeatureName`
3. Compile and run tests locally.

### Coding Standards
- Follow Java naming conventions.
- Maintain CSV header consistency.
- Add clear comments for public methods.
- Ensure email sending uses secure credentials.

### Pull Requests
- Submit PRs to `main` branch.
- Include a short description and issue reference.
- Run all tests before submitting.

### Testing Checklist
- Donor registration & ID generation
- Appointment booking & email confirmation
- Disease-specific flows (Thalassemia/SCD)
- Blood request submission & fulfillment
