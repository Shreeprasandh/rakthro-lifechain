1. PROJECT OVERVIEW
================================================================================

RAKTHRO is a comprehensive Java-based console application designed to streamline blood donation management in Tamil Nadu, India. The system facilitates donor registration, appointment scheduling, blood request handling, and specialized disease-based donation programs for Thalassemia and Sickle Cell Anemia patients. Built with a focus on user-friendly interfaces and robust data management, RAKTHRO serves as a bridge between blood donors and recipients through efficient coordination and communication.

Key Features:
- Donor Management: Complete lifecycle from registration to donation tracking
- Appointment System: Automated booking with hospital selection and email confirmations
- Blood Request Portal: Emergency request submission and fulfillment
- Authentication System: Secure login for admins and users
- Email Notifications: Automated confirmations and updates
- Disease-Based Programs: Specialized support for Thalassemia and Sickle Cell Anemia
- Administrative Panel: System oversight with analytics and monitoring

- db/thal_donors.csv: ThalID,DonorID,Name,... (extensive fields)
- db/thal_checkups.csv: Checkup records
- db/thal_donations.csv: Donation history
- db/thal_blood_requests.csv: Thalassemia requests
- Similar for Sickle Cell (sickle_*.csv)

Request Files:
- db/blood_requests.csv: Name,City,BloodGroup,Contact,Email
- db/requests.csv: Generated analytics (City,BloodGroup,Requests)

Data Relationships:
- DonorID links across all donor-related files
- ThalID/SCDID link to main DonorID
- Email used for notifications and auth
- City-based hospital mapping

Data Flow:
1. User input → Validation → Object creation → Save to CSV
2. Load from CSV → In-memory lists → Operations → Save back
3. Appointments: Generate slip → Log to CSV → Send email
4. Requests: Submit → Store → Fulfill → Remove from CSV

6. APPLICATION FLOW
================================================================================

Startup:
1. Main.main() executes
2. Initialize managers and utilities
3. Display welcome message

Main Loop:
1. Show menu: Admin Login / User Login / Exit
2. Choice 1: Admin Login
   - Prompt username/password
   - AuthSystem.login() checks admin.csv
   - If success, enter admin panel
3. Choice 2: User Login
   - Submenu: Log In / Sign In
   - Log In: By ID or credentials, AuthSystem.userLogin*()
   - Sign In: Register new donor, add to user.csv
   - If success, enter user panel

Admin Panel Loop:
- View All Donors, Search, Edit, Delete, View Appointments, View Requests, etc.
- Each option calls DonorManager or other methods

User Panel Loop:
- Register Donor, Donation Appointment, Dashboard, Disease-Based, Blood Request, Logout
- Appointment: Check eligibility, select hospital, generate slip, email
- Dashboard: City/BG analytics
- Disease: Route to Thalassemia/Sickle Cell menus
- Request: Handle blood requests

Disease Menus:
- Thalassemia: Register, Checkup, Donation, Dashboard, History, Requests
- Sickle Cell: Similar, with transfusion focus

Exit: System.out.println("Exiting..."); break;
